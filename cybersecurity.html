Encrypting Data

Websites with secure domain protocols will use proper encryption to prevent others from accessing private information. If a user is sending information such as passwords or credit card numbers to a website, this information will be encrypted using strings of characters to make it virtually impossible for the information to be stolen. Over time, encryption has been updated to include strings with more characters to decrease the likelihood that sensitive information is decrypted and stolen.

Public versus Private Key

Websites utilize public and private keys for users to not only access their online files privately, but allow others to send private information without fear of such content being stolen. The public key allows as many people as the user wants to send the information, but only the private key allows the user to open and access all of this content.

CyberSecurity Issues

Despite the efforts of international organizations and worldwide governments, CyberSecurity issues are experienced throughout the web. Those committing such crimes will break through weak encryption or faulty software/hardware to send viruses that will weaken systems and make them vulnerable to cyber attacks. In addition, manipulative tactics such as "phishing" emails are used to get users to willfully send private information over the web so that cyber criminals get access to sensitive data, such as bank accounts or social security numbers. While initiatives are in place to try to prevent this, individual users have a responsibility to exclusively stay on websites with secure protocols and to not download content from any websites with questionable security credentials.
